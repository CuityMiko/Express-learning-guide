## 项目目录概览

项目目录结构如下，这里只列出第一层结构。以下分别为

* app.js：项目的主入口，一般都是在这里面做修改，比如新增路由规则等。
* bin：项目启动脚本所在目录。比如启动服务的 **npm start**，其实运行的就是 **bin/start** 脚本。
* public：存放项目的静态文件，比如js、css、img等。
* routes：用来存放项目的路由配置。
* views：存放项目的模板文件。
* node_modules: 项目依赖的node模块，比如各种中间件等。
* package.json：项目依赖声明。


```bash
.
├── app.js    # 项目的主入口，一般都是在这里面做修改，比如新增路由规则等。
├── bin     # 项目启动脚本所在目录。比如启动服务的 **npm start**，其实运行的就是 **bin/start** 脚本。
├── node_modules    # 项目依赖的node模块，比如各种中间件等。
├── package.json    # 项目依赖声明。
├── public    # 存放项目的静态文件，比如js、css、img等。
├── routes    # 用来存放项目的路由配置。
└── views    #存放项目的模板文件。

5 directories, 2 files
```
